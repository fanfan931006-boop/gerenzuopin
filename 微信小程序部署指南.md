# 微信小程序部署指南

## 一、技术栈调整

由于微信小程序使用的是原生框架（WXML + WXSS + JavaScript），而之前的框架使用的是React + Tailwind CSS，需要进行技术栈调整。有以下两种方案：

### 方案1：使用跨端框架（推荐）
- **Taro**：支持使用React语法开发，可编译为微信小程序
- **uni-app**：支持使用Vue语法开发，可编译为微信小程序
- **优势**：代码复用率高，开发效率快

### 方案2：原生开发
- 直接使用微信小程序原生框架开发
- **优势**：性能更好，更贴近小程序原生能力

## 二、微信小程序账号注册

1. **访问微信公众平台**：https://mp.weixin.qq.com/
2. **注册账号**：选择「小程序」类型注册
3. **完善信息**：填写小程序名称、头像、简介等
4. **获取AppID**：在「设置」→「基本设置」中获取AppID

## 三、开发环境配置

1. **下载微信开发者工具**：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. **安装并登录**：使用微信扫码登录
3. **创建项目**：
   - 选择「小程序」项目类型
   - 填写项目名称和目录
   - 输入之前获取的AppID
   - 选择「不使用云服务」（如果需要云开发可选择使用）

## 四、代码迁移

### 使用Taro框架示例

1. **安装Taro**：
   ```bash
   npm install -g @tarojs/cli
   ```

2. **初始化项目**：
   ```bash
   taro init portfolio-app
   ```
   选择React模板

3. **安装依赖**：
   ```bash
   cd portfolio-app
   npm install
   ```

4. **代码迁移**：
   - 将之前的组件逻辑迁移到Taro组件
   - 调整样式为Taro支持的格式
   - 适配小程序API

### 核心功能适配

1. **瀑布流布局**：
   - 使用小程序的`masonry`布局或自定义组件
   - 参考：`wx.createSelectorQuery()`获取元素尺寸

2. **作品上传**：
   - 使用`wx.chooseImage()`选择图片
   - 使用`wx.uploadFile()`上传图片
   - 后端API需要配置为小程序合法域名

3. **扫码查看**：
   - 使用`wx.createQRCode()`生成二维码
   - 或使用第三方库生成二维码

4. **数据统计**：
   - 集成微信小程序数据统计功能
   - 或使用自定义埋点

## 五、后端服务配置

### 方案1：自建服务器

1. **搭建后端服务**：
   - 使用Node.js + Express
   - 配置数据库（MongoDB或MySQL）
   - 部署到云服务器（如腾讯云、阿里云）

2. **配置域名**：
   - 注册域名并备案
   - 配置HTTPS证书
   - 在小程序后台「开发」→「开发设置」中添加合法域名

### 方案2：使用微信云开发

1. **开通云开发**：
   - 在微信开发者工具中开通云开发
   - 创建云环境

2. **云函数**：
   - 编写云函数处理业务逻辑
   - 如作品上传、数据查询等

3. **云存储**：
   - 使用云存储存储作品图片
   - 无需配置域名，更方便

## 六、审核发布流程

1. **本地调试**：
   - 在微信开发者工具中测试功能
   - 确保所有功能正常运行

2. **提交审核**：
   - 点击「上传」按钮，填写版本号和描述
   - 在微信公众平台「版本管理」中提交审核

3. **审核通过**：
   - 审核通过后，在「版本管理」中发布
   - 可选择「全量发布」或「灰度发布」

## 七、注意事项

1. **内容合规**：
   - 确保作品内容符合微信小程序审核规范
   - 避免涉及违规内容

2. **性能优化**：
   - 图片压缩和懒加载
   - 减少首屏加载时间
   - 优化小程序包大小（限制2MB）

3. **用户体验**：
   - 适配不同屏幕尺寸
   - 优化触摸交互
   - 添加加载状态和错误处理

4. **安全措施**：
   - 防止XSS攻击
   - 保护用户数据
   - 后端API添加鉴权

## 八、常见问题

1. **API调用失败**：
   - 检查域名是否已配置
   - 确保HTTPS证书有效

2. **审核被拒**：
   - 查看审核反馈，修改违规内容
   - 确保功能符合小程序规范

3. **图片上传失败**：
   - 检查云存储权限
   - 确保图片大小在限制范围内

## 九、总结

微信小程序部署需要从技术栈调整、账号注册、开发环境配置、代码迁移、后端服务配置到审核发布的完整流程。使用跨端框架如Taro可以提高开发效率，而云开发则可以简化后端部署。遵循微信小程序的规范和最佳实践，确保作品能够顺利上线并提供良好的用户体验。
